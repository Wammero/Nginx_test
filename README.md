Проект разворачивает простое веб-приложение на Python за Nginx reverse proxy, используя Docker Compose.

## Описание архитектуры
Схема работы сервиса (`nginx` → `backend`):

1.  **Nginx (Frontend)**: Слушает порт `80` на хост-машине. Принимает входящие HTTP-запросы и проксирует их внутри Docker-сети.
2.  **Backend (Python)**: Работает на порту `8080`. Принимает запросы от Nginx и возвращает текстовый ответ.
3.  **Безопасность**: Порт бэкенда (`8080`) не опубликован наружу и доступен только контейнеру Nginx внутри изолированной сети `app_network`.

## Как запустить проект

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/Wammero/Nginx_test.git
    cd Nginx_test
    ```

2.  Запустите контейнеры с помощью Docker Compose:
    ```bash
    docker-compose up -d --build
    ```

## Как проверить результат

Для проверки работоспособности выполните команды в терминале:

1. Основная проверка (через Nginx): Выполните команду в терминале для обращения к прокси-серверу:
```bash
curl http://localhost
```
Ожидаемый ответ: Hello from Effective Mobile!

2. Проверка изоляции бэкенда (Безопасность): Попробуйте обратиться к бэкенду напрямую через порт 8080
```bash
curl http://localhost:8080
```
Ожидаемый результат: Ошибка (например, Connection refused или Couldn't connect to server). Это подтверждает выполнение требований безопасности
